# How to Add PHP Code in WordPress (Without Plugins)

This tutorial teaches you how to safely inject PHP code into your WordPress posts, pages, and theme areas without using a plugin. It covers various methods for inserting code into different parts of your site using WordPress hooks and conditional tags.

---

## Introduction

WordPress does not execute PHP code directly in posts or pages for security reasons. Instead, you place your custom PHP code inside your **child theme's `functions.php` file** and use **WordPress hooks** to output it at the desired location.

---

## Important Prerequisite: Use a Child Theme

> **Critical:** You **must** create and use a child theme for your main theme. Otherwise, all customizations in `functions.php` will be lost when the parent theme is updated.
> [Learn how to create a child theme](link-to-your-child-theme-guide)

## Insert PHP Code in Different Areas

### 1. Insert PHP Code in the Frontend (All Pages)

Use the `init` hook with the `!is_admin()` conditional tag to display code on all front-end pages:

```php
// Snippet code in WordPress front-end
add_action( 'init', 'frontEndFunction' );
function frontEndFunction() { 
    if(!is_admin()) {
        // Your PHP code here (e.g., code1)
    }
}
```

### 2. Insert PHP Code in the Backend (Admin Area)

Use the `is_admin()` conditional tag to display code only in the WordPress dashboard:

```php
// Snippet code in WordPress backend
add_action( 'init', 'backEndFunction' );
function backEndFunction() { 
    if(is_admin()) {
        // Your PHP code here
    }
}
```

---

### 3. Insert PHP Code in the `<head>` Section

Use the `wp_head` hook to add code to the `<head>` section of your site:

```php
// Snippet code in WordPress header
add_action('wp_head', 'your_function_name');
function your_function_name(){
    // Your PHP code here
}
```

#### Insert Scripts (e.g., Google Analytics) in the `<head>`

Close the PHP tag (`?>`) to directly output script or HTML, then reopen it:

```php
// Insert script or HTML in WordPress header
add_action('wp_head', 'your_function_name');
function your_function_name(){ ?>

<!-- Put your script or HTML code here -->
<script>
// Your JavaScript or tracking code
</script>

<?php }
```

#### Enqueue Scripts and Styles Properly

For adding external `.js` or `.css` files, use the `wp_enqueue_scripts` hook with `wp_enqueue_style()` and `wp_enqueue_script()` functions:

```php
/**
 * Proper way to enqueue scripts and styles.
 */
function wpdocs_theme_name_scripts() {
    wp_enqueue_style( 'style-name', get_stylesheet_uri() );
    wp_enqueue_script( 'script-name', get_template_directory_uri() . '/js/example.js', array(), '1.0.0', true );
}
add_action( 'wp_enqueue_scripts', 'wpdocs_theme_name_scripts' );
```

> **Tip:** Learn more about [wp_enqueue_script](https://developer.wordpress.org/reference/functions/wp_enqueue_script/) in the official WordPress documentation.

---

### 4. Insert PHP Code After the Opening `<body>` Tag

Use the `wp_body_open` hook (available in most modern themes) to add code immediately after the opening `<body>` tag:

```php
// Snippet code in opening body tag
add_action('wp_body_open', 'your_function_name');
function your_function_name(){
    // Your PHP code here
}
```

---

### 5. Insert PHP Code in the Footer

Use the `wp_footer` hook to add code just before the closing `</body>` tag:

```php
// Snippet code in WordPress footer
add_action('wp_footer', 'your_function_name');
function your_function_name(){
    // Your PHP code here
}
```

#### Insert Scripts in the Footer

Close the PHP tag to output scripts directly:

```php
// Insert script or HTML in WordPress footer
add_action('wp_footer', 'your_function_name');
function your_function_name(){ ?>

<!-- Put your script or HTML code here -->
<script>
// Your JavaScript or tracking code
</script>

<?php }
```

## Insert PHP Code on Specific Pages (Conditional Tags)

WordPress offers **conditional tags** to target specific pages. Here are the most common ones:

### Getting Page/Post IDs

To find a page or post ID:
1. Go to **Pages** or **Posts** in the admin dashboard.
2. Hover your mouse over the page title.
3. Look at the bottom-left corner of your browser; the URL will show `post=XX`, where `XX` is the ID.

### Code Snippet for Specific Posts

Use `is_single()` with optional parameters to target single posts.

```php
// Snippet code in WordPress posts
add_action( 'wp_body_open', 'your_function_name');
function your_function_name(){
    if ( is_single() ) {  // All single posts
    // if ( is_single( '17' ) ) {  // Post with ID 17
    // if ( is_single( 'Black Cat' ) ) {  // Post with title "Black Cat"
    // if ( is_single( array( 17, 'Black Cat' ) ) ) {  // Post ID 17 or title "Black Cat"
        // Your PHP code here
    }
}
```

### Code Snippet for Category Pages

Use `is_category()` to target category archives.

```php
// Snippet code in WordPress Category pages
add_action( 'wp_head', 'your_function_name');

    }

```

### Code Snippet for Tag Pages

## Important Notes and Security

### Is it safe to add custom code directly to functions.php?

While adding custom code to `functions.php` is common, it can pose risks if not done correctly:
- **Always use a child theme** to prevent losing changes during updates.
- **Understand the code** you are adding.
- **Test in a staging environment** before deploying to a live site.

### What if I break my website with custom code?

If you encounter a fatal error (e.g., a "white screen" or syntax error):
1. Access your site via **FTP** or your hosting control panel's file manager.
2. Navigate to the `functions.php` file in your child theme folder.
3. Download a backup of the file, then open it in a text editor.
4. **Remove the custom code you added** and save the file.
5. Re-upload the file to your server.
6. Your site should now be accessible. You can re-add the code carefully.

### Alternative to functions.php

If you prefer not to edit `functions.php` directly, you have other options:
- **Create a custom plugin** for your code snippets (keeps theme files clean).
- Use a code snippet management plugin like **Code Snippets** (allows managing snippets from the dashboard).
